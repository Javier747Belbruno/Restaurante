Scenario Type
Flujo Básico con selección de productos individuales y búsqueda de estos por categorías y subcategorías Basic Path
Details:
Steps:
1. Selecciona la opción Registrar Pedido de Mesa. <br/>
2. Busca para el usuario logueado, las mesas asignadas actualmente que se encuentren abiertas, visualizando para cada mesa los siguientes datos: nro. de mesa, sección, sector y piso, solicitando se seleccione la mesa (Observación 2). <br/>
3. Selecciona una mesa (Observación 3) <br/>
4. Para cada ítem que desee agregarse al pedido, permite elegir si se trata de un menú sugerido o de un producto individual. <br/>
5. Opta por agregar productos individuales.<br/>
6. Solicita la selección de ítems para el pedido, para lo cual permite al Mozo la búsqueda de estos mediante los siguientes criterios: ? Búsqueda de productos por categorías y subcategorías de la carta vigente. ? Búsqueda entre productos configurados como favoritos en la carta vigente.  ? Búsqueda entre todos los productos de la carta vigente mientras ingresa el nombre de estos. <br/>
7. Elige efectuar la búsqueda por categorías y subcategorías. <br/>
8. Muestra las categorías de la carta vigente (Entradas, Platos Principales, Guarniciones, Salsas, Postres, Bebidas), y solicita la selección de una de ellas. <br/>
9. Selecciona una categoría. <br/>
10. Para la categoría seleccionada, muestra las subcategorías correspondientes, solicitando la selección de una de ellas.<br/>
11. Selecciona una subcategoría. <br/>
12. Para cada subcategoría seleccionada, muestra los nombres de los productos incluidos en la misma, y solicita se seleccione el que se desea incluir en el pedido. <br/>
13. Selecciona un producto a incluir en el pedido. <br/>
14. Para cada producto seleccionado, solicita se ingrese la cantidad de este <br/>
15. Ingresa la cantidad para cada producto seleccionado. <br/>
16. Para cada producto ingresado en el pedido de la mesa, busca y muestra el nombre, la cantidad, el precio y el tiempo de presentación de cada producto, permitiendo la modificación de la cantidad y del tiempo de presentación de cualquiera de los productos. <br/>
17. No modifica los productos ingresados. <br/>
18. Solicita confirmación para la registración del pedido.<br/>
19. Confirma la registración. <br/>
20. Registra el pedido (fecha y hora actual, mozo que tomó el pedido, detalle del pedido con cada uno de los productos, sus cantidades y precios) en estado “abierto”, y cada uno de sus detalles en estado “pendiente de preparación”, actualizando el estado de la/s mesa/s correspondiente/s a “con pedido generado”. <br/>
21. Solicita confirmación para imprimir el Pedido.<br/>
22. No confirma la impresión del Pedido. Fin del caso de uso <br/>
Exceptions:
2a. A1: No existen mesas en estado abierta, asignadas al mozo logueado. (at End)
A1: No existen mesas en estado abierta, asignadas al mozo logueado. Exception
